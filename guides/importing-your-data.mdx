title: Importing Your Data
description: A guide to migrating your existing customer, product, inventory, and order data into Caska using CSV files — with tips on order, format, and what to do when rows are skipped.
---

If you're coming from a spreadsheet or another system, this guide covers how to get your existing data into Caska cleanly.

## Recommended import order

Import in this sequence to avoid reference issues:

1. **Customers** — needed before importing orders
2. **Inventory** — needed before building product recipes
3. **Products** — needed before importing orders
4. **Orders** — import last, after customers and products exist

## General import process

The process is the same for each module:

<Steps>
  <Step title="Download the sample template" titleType="p">
    Go to the relevant module list (e.g. Customers), click **Import**, and download the sample CSV template. This shows you the exact column names and format Caska expects. If you have set up custom fields alread, these will display as columns in the CSV.
  </Step>

  <Step title="Format your data" titleType="p">
    Map your existing data to the template columns. Key formatting rules:

    - First row must be column headers matching the template
    - Dates in `YYYY-MM-DD` format
    - UTF-8 encoding (the default when saving as CSV from Excel or Google Sheets)
    - Extra columns are ignored

  </Step>

<Step title="Upload and review" titleType="p">
  Click **Import**, upload your file, and review the import summary. You'll see
  how many records were created and which rows were skipped with reasons.
</Step>

  <Step title="Fix and re-import skipped rows" titleType="p">
    Correct any problem rows in a new CSV and re-import. Caska skips bad rows rather than failing the whole file, so your clean data always gets through.
  </Step>
</Steps>

## Module-specific notes

**Customers**

- Required field: Company Name
- Categories must already exist in Settings before they'll match on import
- Custom fields must already exist in Settings before they'll match on import

**Products**

- Required field: Product Name
- Recipes and ingredient links cannot be imported via CSV — add these manually after import
- Custom fields must already exist in Settings before they'll match on import

**Inventory**

- Required field: Item Name
- Custom fields must already exist in Settings before they'll match on import

**Orders**

- Required field: Customer Name
- Customer names must exactly match existing customer records to link correctly
- Custom fields must already exist in Settings before they'll match on import

<Callout kind="tip">
  For large imports (thousands of records), split your CSV into batches of
  500–1000 rows. Smaller files are easier to troubleshoot if something goes
  wrong.
</Callout>

## After importing

- Review each module list to spot any obvious issues
- For orders, verify customer associations are linked correctly
- Set up product recipes manually after importing products and inventory
