---
title: Communication Settings
description: Configure a custom SMTP server so Caska can send customer-facing emails through your own email address and domain.
---

Caska can send emails on your behalf using your own email server. This is optional — if you don't configure custom SMTP, Caska uses its own email system for all outbound messages.

## Setting up custom SMTP

<Steps>
  <Step title="Go to Settings → Communication" titleType="p">
    Find the Communication section in your account settings.
  </Step>

  <Step title="Enter your SMTP server details" titleType="p">
    - Host (e.g. `smtp.gmail.com`, `smtp.mailgun.org`)
    - Port (typically `587` for TLS or `465` for SSL)
    - Username and password
    - From name and from email address
  </Step>

  <Step title="Save and send a test email" titleType="p">
    Click **Send Test Email** to verify the connection is working before relying on it for customer emails.
  </Step>
</Steps>

## Troubleshooting

If test emails aren't arriving:

- Double-check your SMTP credentials
- Make sure your email provider allows third-party SMTP access
- Gmail users need an **App Password** rather than their regular account password — standard Gmail passwords won't work with third-party SMTP

<Callout kind="info">
  The Follow-Up (re-engagement) system always sends via Caska's system email (Mailgun), regardless of your custom SMTP setting. This ensures reliable automated delivery.
</Callout>
